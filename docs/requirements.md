# 个人技术交流平台｜需求文档（ViteTS\+Gin\+PostgreSQL\+Redis）

## 一、项目概述

本项目为**个人技术交流分享平台**，面向嵌入式、硬件开发、单片机技术爱好者，用于个人技术沉淀、实操经验分享、技术图文交流。整体采用前后端分离架构，基于 Vite\+Vue3\+TypeScript 前端、Golang Gin 后端、PostgreSQL 主业务数据库、Redis 缓存中间件搭建。

平台核心定位：**专注嵌入式与硬件开发领域的技术博客与交流站点**，同时架构预留充足拓展能力，支持后续迭代社区、问答、资源下载、作品集等业务场景。

## 二、技术栈规范（固定不变）

### 前端

- 核心框架：Vite \+ Vue3 \+ TypeScript（**全TS严格类型开发**）

- 路由：VueRouter4

- 状态管理：Pinia

- UI框架：Element Plus

- 网络请求：Axios（二次封装泛型TS）

- 工程化：ESLint \+ Prettier \+ 路径别名 \+ 环境变量区分环境

- 特色能力：**全站中英文双语国际化**

### 后端

- Web框架：Gin（Golang）

- 主数据库：PostgreSQL（稳定、适合博文、长文本、结构化数据）

- 缓存中间件：Redis（热点文章、访问量、限流、会话、缓存）

- ORM：GORM v2

- 认证体系：JWT

- 架构：分层架构（controller/service/model/middleware）高可拓展

## 三、业务核心需求

平台内容**垂直聚焦硬件与嵌入式开发方向**，涵盖三大核心内容板块：

### 1\. 嵌入式 Linux 技术分享

- Linux 环境搭建、驱动开发、系统移植实操记录

- Shell、Makefile、设备树、内核调试经验

- 嵌入式项目实战、踩坑记录、工程经验总结

### 2\. 硬件电路设计经验

- 原理图设计、PCB 布局、硬件调试技巧

- 电源设计、信号完整性、硬件问题排查复盘

- 硬件项目实操图文记录

### 3\. 单片机开发实操分享

- STM32、51、ESP32 等单片机开发教程

- 外设驱动、通信协议、项目实战案例

- 底层代码、实操调试、项目落地经验分享

### 4\. 全站图文内容体系

- 支持富文本图文发布、编辑、归档、分类、标签

- 文章浏览统计、热门文章、最新文章

- 内容支持后续拓展：评论、点赞、收藏、资源附件

## 四、特色功能需求（重点）

### 1\. 全站中英文双语国际化

- 前端页面、菜单、按钮、提示语、系统文案**全部支持中英切换**

- 文章支持：单独中文内容 / 单独英文内容 / 双语内容

- 记住用户语言偏好，全局生效

### 2\. 高可拓展架构设计（核心要求）

整体前后端架构**必须预留业务拓展能力**，满足未来新增场景，包括但不限于：

- 技术问答社区模块

- 资源下载、源码分享、固件下载

- 项目作品集展示

- 留言、评论、互动体系

- 权限后台、内容管理后台

实现方式：模块化路由、TS严格类型、后端分层解耦、通用基础表结构、通用返回体封装。

## 五、通用基础功能需求

### 前端用户端

- 首页展示：分类推荐、最新技术文章、热门文章

- 技术专栏分类：Linux嵌入式 / 硬件电路 / 单片机

- 文章详情、阅读统计、标签筛选、搜索功能

- 全局中英文切换

- 响应式适配电脑/手机端

### 后端服务能力

- JWT 登录认证、后台内容管理

- 文章 CRUD、分类管理、标签管理

- Redis 缓存文章列表、访问量、热门数据

- PostgreSQL 存储海量博文、长文本、结构化数据

- 统一接口返回格式、全局异常处理、参数校验

## 六、架构设计约束（为了长期拓展）

1. **前端全部TS严格模式**：禁止任意any，所有业务数据、接口、路由、状态全部强类型，方便后续迭代大型功能。

2. **业务完全解耦**：页面、组件、hooks、api、types 分层清晰，新增业务直接新增文件夹，不改动旧代码。

3. **后端标准分层**：model、service、controller、router、middleware 完全分层，方便后续拓展权限、评论、资源、会员体系。

4. **双语体系独立**：语言包统一管理，新增页面无需重构双语逻辑。

5. **缓存与数据库分离**：热点数据Redis缓存，业务数据PG持久化，保证后续高访问量稳定。
